# Korean Law (한국 법령·판례 검색)

법제처 OpenAPI 기반 `korean-law` MCP 서버의 도구로 한국 법령·판례를 조사한다.
데이터는 사내 fork(ib-korean-law-mcp) 코드에서 직접 실행되며 fly.dev 중계를 거치지 않는다.

## 사용 절차

1. **식별자 먼저 확보** — 법령을 조회하기 전 `search_law`로 법령명을 검색해 `mst`(또는 lawId)를 얻는다. 약칭도 자동 변환된다.
2. **조문 원문** — `get_law_text(mst=..., jo="제N조")` 로 특정 조문 전문을 가져온다. `jo` 생략 시 전체.
3. **판례/결정문** — `search_decisions(domain=..., query=...)` 로 검색. 도메인 예: 대법원 판례, 헌재, 행정심판, `pipc`(개인정보위), `nts`(국세청), 공정위 등 18종. 키워드는 **공백=AND**라 많을수록 결과가 준다 → 실패 시 키워드를 줄여 재시도.
4. **심화** — 법체계 3단 비교·처분 근거 추적·개정 이력·인용 검증 등은 `legal_research`(task 지정), 정밀 분석은 `legal_analysis` 사용.
5. 도구가 안 보이거나 더 필요한 기능은 `discover_tools` 로 탐색, `execute_tool` 로 실행.

## 원칙

- **환각 금지**: 법령명·조문번호·판례 사건번호를 추측하지 말고 반드시 도구 결과로 확인한다. 도구가 "결과 없음"을 반환하면 그대로 보고하고, 지어내지 않는다.
- **현행 여부 확인**: 조회 결과의 시행일·조회기준일을 확인하고, 개정 의심 시 `search_law`로 [현행] 버전을 재확인한다.
- 출처(법령명, 조문, 시행일, 사건번호)를 답변에 함께 제시한다.

## 요건

MCP 서버가 연결돼 있어야 한다(node·npx·git 필요). 최초 사용 시 `LAW_OC`(법제처 API 키) 입력이 요구된다.
